Java程序异常排查

发表于 2021-09-27 分类于 Java

Java程序异常排查的一些基本步骤。

CPU占用过高排查

CPU 占用过高的原因通常有以下几个：

  • 线程中有比较耗时的计算
  • 系统发生了频繁的 Full GC

排查步骤

  1. 通过 top 命令找到 CPU占用高的 Java 应用程序

  2. 使用 top -H -p PID 查看对应进程的哪个线程占用CPU过高，例如：top -H -p 22092

  3. 使用 printf "%x\n" tid 将 CPU 占用高的线程 ID 转换为16进制格式

  4. 使用 jstack pid >> thread.txt 导出线程堆栈信息，例如：jstack 22092 >> thread.txt

  5. 从线程堆栈信息中查找 CPU 占用高的16进制线程 ID，找到对应的线程栈，分析线程栈信息

内存溢出排查

JVM 发生 OOM 时自动生成 hprof 文件 -> 增加启动参数：-XX:+HeapDumpOnOutOfMemoryError -XX:HeapDumpPath=./logs

  • -XX:+HeapDumpOnOutOfMemoryError：表示当JVM发生OOM时，自动生成 hprof 文件。
  • -XX:HeapDumpPath=${目录}：表示生成DUMP文件的路径，也可以指定文件名称。

GC日志分析

在 Java 程序启动脚本中加入：

选项：

  • -XX:+PrintGC：输出GC日志
  • -XX:+PrintGCDetails：输出GC的详细日志
  • -XX:+PrintGCTimeStamps：输出GC的时间戳（以基准时间的形式）
  • -XX:+PrintGCDateStamps：输出GC的时间戳（以日期的形式，如 2013-05-04T21:53:59.234+0800）
  • -XX:+PrintHeapAtGC：在进行GC的前后打印出堆的信息
  • -Xloggc:../logs/gc.log：GC日志文件的输出路径

远程监控 Java 程序

在需监控的远程 Java 程序启动脚本中加入：

选项：

  • -Dcom.sun.management.jmxremote=true：表示开启JVM远程监控
  • -Djava.rmi.server.hostname=10.86.52.68：设置监控的 IP
  • -Dcom.sun.management.jmxremote.port=28080：设置监听的端口
  • -Dcom.sun.management.jmxremote.ssl=false：设置是否启用 SSL
  • -Dcom.sun.management.jmxremote.authenticate=false：设置是否需要用户密码验证
